\section{Pair Programming}

\subsection{O que é Pair Programming} 

\begin{frame}{\textit{Pair Programming} (Programação em Par)}

\begin{itemize}
\item Técnica de programação baseada na metodologia ágil \textit{Extreme Programming (XP)}\footnote{\url{http://www.extremeprogramming.org/rules/pair.html}}.
\end{itemize}

\begin{block}{Pair Programming (Programação em Par)}
Todo código a ser enviado para produção foi criado por duas pessoas trabalhando juntas em um único computador. \textit{Pair programming} aumenta a qualidade do software sem impactar no tempo para a entrega.
\end{block}
\end{frame}

\begin{frame}{Como implementar a Programação em Par}
Regras:

\begin{itemize}
   \item Alternância de papeis, entre o piloto e copiloto.
   \item Atitude colaborativa. Um codifica e o outro supervisiona se os padrões de projeto e princípios de qualidade estão sendo aplicados. 
   \item Não é \textit{mentoring}. Deve-se tentar ideias de ambos os membros da equipe.
   \item Piloto deve sempre falar em voz alta o que está fazendo.
\end{itemize}
\end{frame}

\subsection{Como implementar a Programação em Par} 

\begin{frame}{Como implementar a Programação em Par}
Passos:

\begin{enumerate}
\item Defina bem a tarefa antes de começar.
    \begin{itemize}
        \item Deve ser possível completá-la em até 2 horas.
        \item Ex: Escrever comentário, ou adicionar código de manutenção.
    \end{itemize}
\item Objetivos pequenos para definir o foco da concentração.
    \begin{itemize}
        \item Verbalizar o objetivo ajuda a concentrar.
        \item Pequenos objetivos devem ser alcançáveis em questão de minutos.
    \end{itemize}
\item Use TDD (\textit{Test-driven development})
    \begin{itemize}
        \item Primeiro escreva o teste unitário do micro objetivo.
        \item Objetivo seguinte deve ser fazer o código passar no teste.
    \end{itemize}
\end{enumerate}
\end{frame}

\begin{frame}{Como implementar a Programação em Par}
Passos (continuação):

\begin{enumerate}\setcounter{enumi}{3}

\item Confie e apoie o seu parceiro.
    \begin{itemize}
        \item Quando estiver pilotando, tente desenvolver o mais rápido possível.
        \item Confie no observador para apontar os erros e te dar segurança.
        \item Observador deve ler o código enquanto é desenvolvido.
        \item Erros e trechos de código difíceis de entender devem ser corrigidos imediatamente,
        \item Deixe problemas maiores e ideias de melhoria para depois.
        \item Observador não pode ditar código.
    \end{itemize}
\end{enumerate}
\end{frame}

\begin{frame}{Como implementar a Programação em Par}
Passos (continuação):

\begin{enumerate}\setcounter{enumi}{4}

\item Mantenha a sincronia.
    \begin{itemize}
        \item Diga o que vai fazer antes de fazer.
        \item Se o parceiro pedir opinião, concordar ou discordar imediatamente.
    \end{itemize}
\item Seja cordial ao apontar erros.
\item Comemore ao terminar uma tarefa.
\item Troquem de papeis com frequência, de aproximadamente meia hora.
\end{enumerate}
\end{frame}
